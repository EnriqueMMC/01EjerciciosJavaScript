JavaScript

Comentar el código

  • Para comentar una línea de código se utiliza el símbolo // al inicio de la línea
  • Para comentar varias líneas de código se utiliza el símbolo /* al inicio y */ al final

¿Por qué comentar el código?

  • Para que el código sea más legible
  • Para que otros programadores puedan entenderlo
  • Para que tú mismo puedas entenderlo en el futuro
  • Para que el código sea más fácil de mantener
  • Para que el código sea más fácil de depurar
  • Para que el código sea más fácil de modificar

¿Qué es la consola de JavaScript?

  • La consola de JavaScript es una herramienta que permite ver los mensajes de error y advertencia que se producen en el código
  • La consola de JavaScript también permite ejecutar código JavaScript directamente
  • La consola de JavaScript se puede abrir en la mayoría de los navegadores web presionando la tecla F12
  • La consola de JavaScript se puede utilizar para depurar el código
  • La consola de JavaScript se puede utilizar para ver el valor de las variables

Variables

  • Una variable es un espacio en memoria que se utiliza para almacenar un valor
  • Las variables se pueden utilizar para almacenar cualquier tipo de dato
  • Las variables se pueden declarar utilizando la palabra reservada var, let o const
  • Las variables se pueden inicializar con un valor utilizando el símbolo =

Diferencia entre var, let y const

  • var: se puede redeclarar y reasignar
  • let: se puede reasignar pero no redeclarar
  • const: no se puede redeclarar ni reasignar
  • const no se puede utilizar sin inicializar
  • let y const son más recomendables que var

Palabras reservadas

  • Las palabras reservadas son palabras que no se pueden utilizar como nombres de variables
  • Las palabras reservadas son palabras que tienen un significado especial en JavaScript
  • Las palabras reservadas son palabras que no se pueden utilizar como nombres de funciones

Reglas para los nombres de las variables

  • Los nombres de las variables deben comenzar con una letra, un guion bajo (_) o un signo de dólar ($)
  • Los nombres de las variables no pueden comenzar con un número
  • Los nombres de las variables no pueden contener espacios
  • Los nombres de las variables son sensibles a mayúsculas y minúsculas (case-sensitive)
  • Los nombres de las variables no pueden ser palabras reservadas

Estilos para los nombres de variables con más de una palabra

  • camelCase: La primera palabra comienza con minúscula y las siguientes palabras comienzan con mayúscula (ejemplo: miVariableEjemplo)
  • snake_case: Todas las palabras están en minúsculas y se separan con guiones bajos (ejemplo: mi_variable_ejemplo)
  • PascalCase: Todas las palabras comienzan con mayúscula (ejemplo: MiVariableEjemplo)
